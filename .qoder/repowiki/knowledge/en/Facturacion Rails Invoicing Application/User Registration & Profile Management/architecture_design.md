Built on top of Devise (`config/initializers/devise.rb`) with the `User` model enabling `database_authenticatable`, `registerable`, `recoverable`, `rememberable`, `validatable`, `confirmable`, `lockable`, `timeoutable`, and `trackable`. Custom `RegistrationsController < Devise::RegistrationsController` intercepts `update` to build/update the associated `UserProfile` alongside the account, overrides `after_sign_up_path_for` to land users in the multi-step wizard, and uses `update_without_password` when no password is supplied. The `User` model owns a `has_one :user_profile` (with `inverse_of`) and drives step-based validation via an `mattr_accessor form_steps` array (`sign_up → freelance_or_company → set_name → set_address`) plus `required_for_step?` callbacks that conditionally validate the associated profile. A separate `UserProfileController` provides full CRUD over profiles scoped through `current_user.UserProfile`. Views are split between Devise's default templates under `app/views/devise/` (sessions, passwords, confirmations, unlocks) and the application's own `app/views/after_register/` wizard screens (`freelance_or_company`, `set_name`, `set_address`, `set_company_info`) plus shared partials (`_account.html.erb`, `_address.html.erb`, `_personal.html.erb`, `_preferencias.html.erb`, `_sidebar.html.erb`). Localization for the registration flow lives in `config/locales/registration/{en,es}.yml`. Dependency direction: controllers depend on models; views render both Devise-scoped and app-scoped templates; no cross-dependency between `UserProfileController` and `RegistrationsController` beyond sharing the `user_profile_params` permit list.
