The root is a standard Rails 7.2 application (Rack entry in `config.ru`, Puma server via Procfile) whose behaviour is assembled by the `configuration` module's initializers and boot sequence. Feature modules (`clients_feature`, `invoices_feature`, `items_feature`, `line_items_feature`, `dashboard_charts`, `users_auth`, `home_pages`) are loaded as autoloaded directories under `app/` and share cross-cutting concerns defined in `shared_infrastructure` (base controllers/models/helpers, authentication, i18n, pagination, Heroicons, session-backed invoice). The frontend asset pipeline in `frontend_assets` composes Turbo, Stimulus, Alpine.js, Tailwind CSS and ApexCharts on top of Sprockets/importmap, while `database_schema` provides the PostgreSQL schema and seeds consumed by `tests`. All features depend on the shared base classes rather than on each other, keeping feature boundaries isolated at the controller/model level.
