Standard Rails boot chain: `config/boot.rb` pins Bundler and Bootsnap, `config/application.rb` loads `rails/all`, calls `Bundler.require(*Rails.groups)`, and defines `Facturacion::Application` with global defaults (time zone Madrid, autoload_lib, i18n raise-on-missing). `config/environment.rb` is the single entry point that initializes the app. Per-environment overrides live in `config/environments/{development,test,production}.rb`, which take precedence over `application.rb`. Third-party gem configuration is split into one initializer per dependency under `config/initializers/` (Devise, SimpleForm, Pagy, RouteTranslator, Carmen, Heroicon, CSP/permissions policies, form errors, inflections), keeping each concern isolated. Routing is centralized in `config/routes.rb` using Devise's `devise_for :users` plus RESTful resources for users, clients, items, invoices (with nested line_items and a collection action `add_item`) and a mounted `LetterOpenerWeb::Engine`. Runtime/runtime-adjacent settings are externalized to YAML files consumed at boot: `database.yml` (PostgreSQL, env-driven production credentials), `storage.yml` (Disk service per environment, S3/GCS/Azure commented out), `cable.yml` (async/test/redis adapters), and `puma.rb` (thread pool, port, pidfile, tmp_restart plugin).
