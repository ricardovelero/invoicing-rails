The module is split across three Rails conventions:
- `app/javascript/` — ES modules loaded via `importmap-rails`. `application.js` boots `@hotwired/turbo-rails`, `@rails/request.js`, and Alpine (with its Turbo adapter), then imports `controllers/index.js`. That index file is auto-generated by `bin/rails stimulus:manifest:update` and registers every controller under `app/javascript/controllers/*_controller.js` with a kebab-case name derived from the filename.
- `config/importmap.rb` — pins runtime dependencies (`turbo`, `stimulus`, `stimulus-loading`, `tailwindcss-stimulus-components`, `apexcharts`, `alpinejs`, `alpine-turbo-drive-adapter`) and uses `pin_all_from` so new controllers are picked up without manual edits.
- `app/assets/` — Sprockets manifest (`assets/config/manifest.js`) links images, stylesheets, JS, vendor JS, and prebuilt Tailwind output in `builds/tailwind.css`; `stylesheets/application.tailwind.css` is the Tailwind entry consumed by the build.
- `config/tailwind.config.js` extends Tailwind with custom shadows, font sizes, breakpoints, z-index, and a `sidebar-expanded` variant; content scanning covers ERB/HAML/SLIM views, helpers, and JS files.
Dependency direction: controllers depend on Stimulus/Turbo/Alpine only through importmap pins; the asset pipeline depends on the prebuilt `builds/tailwind.css` rather than importing Tailwind at runtime.
