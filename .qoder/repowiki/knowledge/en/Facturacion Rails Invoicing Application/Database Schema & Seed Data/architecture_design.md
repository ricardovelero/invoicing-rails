- `db/schema.rb` is the canonical source of truth (version `2023_12_24_164322`, Rails 7.1) defining six core tables — `users`, `user_profiles`, `clients`, `items`, `invoices`, `line_items` — plus Active Storage extension tables (`active_storage_blobs`, `active_storage_variant_records`) and two extensions: `plpgsql` and `unaccent`.
- `db/migrate/` holds the full evolution history: initial domain tables (`create_items`, `create_clients`, `create_invoices`, `create_line_items`), Devise user scaffolding (`devise_create_users`), a large batch of column-renaming / constraint-fixing migrations in late September 2022 (e.g. `fix_client_column_names`, `remove_invoice_status_column`), profile extraction into `user_profiles` with a one-to-one link back to `users`, and later cleanup dropping legacy Pay gem tables (`drop_pay_tables`, `drop_pay_merchants_table`).
- `db/seeds.rb` is a thin dispatcher that delegates to environment-specific files; currently only `db/seeds/development.rb` exists, which truncates the DB via `database_cleaner`, creates two confirmed users with Spanish `UserProfile`s, then bulk-inserts ~800 synthetic records (clients, items, invoices, line items) using Faker and finally recomputes invoice totals via model callbacks.
